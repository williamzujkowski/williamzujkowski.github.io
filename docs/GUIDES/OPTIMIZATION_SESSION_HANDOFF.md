## üéì Lessons Learned (For Future Reference)

### Estimation Accuracy

**Key Insight:** Must distinguish between:
- **Redundant guidance** (exact duplicates) ‚Üí consolidate aggressively
- **Strategic overlap** (similar content, different context) ‚Üí consolidate with cross-refs
- **Contextual mentions** (brief references) ‚Üí keep separate (NOT redundancy)

**Example:**
- Initial estimate: 4,808 tokens redundant (concurrent execution)
- Actual redundancy: 172 tokens (3.6% accuracy)
- Reason: Conflated "mentions" with "duplicates"

**Corrected methodology:**
1. Line-by-line analysis before consolidation
2. Distinguish exact duplicates from mentions
3. Validate with prototype on sample
4. Use conservative estimates (0.5x multiplier)

**Result:** 85% estimation accuracy on subsequent phases

---

### Swarm Coordination Patterns That Worked

**Most Effective:**
- ‚úÖ **Parallel agent execution** (3 agents simultaneously ‚Üí 3x faster)
- ‚úÖ **Specialized roles** (Researcher, Coder, Tester, Documenter ‚Üí 85-95% confidence)
- ‚úÖ **Consensus validation** (4 agents independently validated ‚Üí caught errors early)
- ‚úÖ **Phased implementation** (validation after each phase ‚Üí zero rollbacks)

**Less Effective:**
- ‚ö†Ô∏è Sequential agent spawning (10-20x slower than parallel)
- ‚ö†Ô∏è Over-estimated redundancy (wasted analysis time)
- ‚ö†Ô∏è Implementation without detailed plan (higher risk)

**Documented in:** `docs/GUIDES/SWARM_COORDINATION_PATTERNS.md`

---

### Architecture Decisions That Worked

**What Worked:**
- ‚úÖ **Lazy loading** (97.9% MANIFEST reduction)
- ‚úÖ **Single source of truth** (easier maintenance)
- ‚úÖ **Progressive disclosure** (load only what you need)
- ‚úÖ **Hash-based validation** (12x faster than O(n¬≤))
- ‚úÖ **Archive, don't delete** (instant rollback capability)

**What Changed:**
- Token estimates need measured data (not theoretical calculations)
- Strategic overlap ‚â† redundancy (preserve workflow context)
- Pre-commit hooks need rename detection (bug fix critical)
