# Agent Rules for LLM Code Generation

This file defines specific expectations and guidelines for AI agents (like Claude) when working with this codebase.

## Code Generation Rules

1. **Consistency**: Generated code must match existing code style and patterns
2. **Documentation**: Include JSDoc or appropriate documentation for all significant functions
3. **Testing**: Consider testability in all code modifications
4. **Performance**: Optimize for web performance, especially for client-side code
5. **Security**: Follow security best practices for web applications

## Response Guidelines

1. **Clarity**: Provide clear explanations for implementation decisions
2. **Context**: Consider the broader project context when suggesting changes
3. **Alternatives**: When appropriate, suggest alternative approaches with pros/cons
4. **Learning**: Include helpful insights about technologies or patterns used

## Ethical Considerations

1. **Accessibility**: Ensure code changes maintain or improve accessibility
2. **Privacy**: Do not suggest code that collects unnecessary user data
3. **Performance**: Consider the environmental impact of computationally expensive operations
4. **Inclusivity**: Use inclusive language in comments and documentation
